# Business Rules — Reinvention Services navigation

> Functional rules that govern the page's expected behaviour. No implementation detail, no test-design guidance — the rules only.

## Sub-navigation
- **BR-1.** The sub-nav lists the page's major sections, in document order (top-to-bottom of the page).
- **BR-2.** The sub-nav is persistent/sticky while the user scrolls, so navigation is always reachable.
- **BR-3.** Selecting a sub-nav item takes the user to that item's section (in-page navigation), and that section's heading is shown.
- **BR-4.** Sub-nav items give a visual affordance on hover, signalling they are actionable.

## Content
- **BR-5.** Every section is introduced by a heading that identifies it (see the item→heading map in `application-overview.md`).
- **BR-6.** The set and order of sections is content-managed and can change over time.
